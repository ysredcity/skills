# UX Base Specification

> 本文档基于海信集团信息化项目 UX 设计规范整理。
> 目标：为 AI 和开发者提供可执行的页面生成规则，确保所有页面风格统一。

---

## 设计价值观

- **一致性**：保持设计的一致性和可预测性，降低用户的学习成本
- **效率**：通过合理的布局和交互设计，提升操作效率
- **清晰**：信息层级清晰，引导用户顺畅地完成任务
- **可控**：给予用户足够的控制权和反馈

## 响应式适配

**布局断点规范**：

- 移动端：< 768px
- 平板端：768px – 1024px
- 桌面端：> 1024px（主要设计基准为 1280px）
- 优先采用移动优先（Mobile First）策略：`@media (min-width: 768px) { ... }`

**弹性布局要求**：

- 页面主内容区域必须使用 `flex: 1` 或 `width: 100%` 自适应父容器宽度，禁止写死固定宽度
- 表格列宽优先使用百分比或 `min-width`，移动端表格允许横向滚动（`overflow-x: auto`）
- 弹窗宽度使用固定值（如 `520px`、`800px`），但不超过视口宽度的 90%：`:style="{ maxWidth: '90vw', width: '100%' }"`
- 侧边栏在移动端应默认收起或切换为 Drawer 模式

**移动端适配要点**：

- 触摸目标最小尺寸不低于 44×44px
- 表单字段在移动端使用单列布局，桌面端使用多列布局
- 操作按钮在移动端优先收折到底部操作栏或"更多"菜单
- 列表页筛选面板在移动端默认折叠
- 字体大小在移动端不低于 14px

## 无障碍（可访问性）

- 颜色不能作为传递信息的唯一手段，必须同时提供文字或图标辅助说明
- 图标按钮必须提供文字说明，使用 `<a-tooltip>` 包裹或设置 `aria-label` 属性
- 表单字段必须通过 `<a-form-item>` 的 `label` 属性提供可见标签，禁止仅用 `placeholder` 代替标签
- 表格必须有列标题（`columns` 中设置 `title`）
- 弹窗打开时焦点应自动移入弹窗内部，关闭后焦点返回触发元素
- 禁止使用 `outline: none` 移除焦点样式

## 全球化兼容

- 表单 label 推荐使用上下布局（label 在上，输入框在下）
- 按钮文字不要写死宽度，使用 `min-width` 保证最短可用宽度
- 布局中避免使用 `float: left/right`，优先使用 Flexbox/Grid

## 设计误区速查

| 错误做法 | 正确做法 |
|------|----------|
| 硬编码色值（如 `color: #00AAA6`） | 使用主题包中的变量（如 `var(--color-primary-6)`） |
| 主按钮超过 1 个 | 每个区块最多 1 个 primary 按钮 |
| 对话框缺少关闭方式 | 对话框必须有取消/关闭操作按钮 |
| 列表操作按钮过多 | ≤3 个时平铺，>3 个时折叠到"更多"下拉菜单 |
| 表单字段过多不分组 | 字段过多时增加分组标题分区展示 |
| 缺少 placeholder | 输入框默认"请输入"，选择器默认"请选择" |
| 必填字段未标记 | 使用 `required` 属性 |
| 危险操作不进行二次确认 | 删除等危险操作必须通过 confirm 对话框二次确认 |
| 筛选器不支持重置 | 查询 + 重置必须成对出现 |
| 空状态空白 | 使用 `<a-empty>` 图标 + 文字占位提示 |
