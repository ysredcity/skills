# 设计系统：Pangea 3 Linear（海信企业级）

## 1. 视觉主题与氛围

Pangea 3 Linear 是海信集团面向内部管理平台的企业级设计系统，基于 Arco Design Vue 构建，搭配自定义的青绿色品牌标识。整体观感呈现出一种**专业克制**的气质：以纯白画布（`#ffffff`）为底，通过精心校准的灰阶体系建立信息层级，而非依赖装饰性元素。这是一套为**数据密集型工作流**量身打造的系统——表单驱动的 CRUD 页面、可筛选的数据表格、多层级导航——在这里，清晰与效率的优先级高于视觉表现力。

品牌色是一种独特的**青绿色**（`rgb(0, 170, 166)` / `--primary-6`），使 Pangea 区别于 Arco Design 默认的蓝色。这是一种冷暖平衡的色调——既不过暖也不过冷——传递出现代感与可信赖感，同时避免了纯蓝色的企业冷感。青绿色的使用非常克制：仅出现在主按钮、选中菜单项、激活态和关键交互元素上。界面的其余部分完全建立在中性灰阶之上，带有微妙的暖色调（`rgb(29, 33, 41)` 用于主文字，`rgb(247, 248, 250)` 用于浅填充）。

字体系统使用 Inter 渲染拉丁字符，PingFang SC / Microsoft YaHei 渲染中日韩文字，并采用针对数据密集界面优化的务实字号体系：**14px 作为默认正文字号**（而非 16px），13px 用于表格单元格和次要文本，12px 用于标签和辅助文字。标题字号也很克制——24px 用于页面标题，20px 用于区块标题——体现了系统对内容密度而非视觉冲击力的优先考量。字重遵循三级模型：Regular（400）用于正文，Medium（500）用于标签和控件，SemiBold（600）用于标题。

阴影系统极简且功能导向：三级 box-shadow 均使用 `rgba(0,0,0,0.1)` 透明度，通过递增的模糊半径（5px、10px、20px）区分层级，另有一个特殊微阴影（`0 0 1px rgba(0,0,0,0.3)`）用于精细的类边框效果。层级感主要通过背景填充色阶梯（`--color-fill-1` 到 `--color-fill-4`）而非阴影深度来传达，保持界面扁平且聚焦。

**核心特征：**
- 青绿色品牌色（`rgb(0, 170, 166)`）替代 Arco Design 默认蓝色
- 明亮干净的画布，`#ffffff` 背景搭配中性灰层级体系
- Inter + PingFang SC / Microsoft YaHei 字体栈，适配中英文混排
- 数据密集优化：14px 默认正文、13px 表格文字、12px 标签
- 保守的圆角：2px（小）、4px（标准）、8px（卡片/弹窗）
- 极简阴影系统——通过背景填充色阶梯而非阴影深度表达层级
- 标准企业布局：48px 顶栏 + 200px 可折叠侧边栏 + 内容区
- `rgb(242, 243, 245)`（fill-2）作为外壳背景，白色用于内容区
- 内容区使用 `border-radius: 8px 0 0 0` 左上圆角，与侧边栏形成视觉分离

## 3. 排版规则

### 字体族
- **拉丁字符**：`Inter`
- **中日韩与系统回退**：`-apple-system, BlinkMacSystemFont, PingFang SC, Hiragino Sans GB, noto sans, Microsoft YaHei, Helvetica Neue, Helvetica, Arial, sans-serif`
- **无自定义 OpenType 特性**——全部使用标准渲染。

### 字号层级

| 角色 | 字号 | 字重 | 行高 | 字间距 | 说明 |
|------|------|------|------|--------|------|
| 运营大标题 | 56px | 600 | 1.5715× | normal | 营销/运营场景的超大标题（企业后台中罕见） |
| 运营中标题 | 48px | 600 | 1.5715× | normal | 大型运营标题 |
| 运营小标题 | 36px | 600 | 1.5715× | normal | 功能区块标题 |
| 页面标题 | 24px | 600 | 1.5715× | normal | 页面级标题 |
| 区块标题 | 20px | 600 | 1.5715× | normal | 区块标题、弹窗标题 |
| 卡片标题 | 16px | 600 | 1.5715× | normal | 卡片标题、列表项标题 |
| 默认正文 | 14px | 400 | 1.5715× | normal | **标准正文**——最常用的字号 |
| 密集正文 | 13px | 400 | 1.5715× | normal | 表格单元格、密集信息展示 |
| 标签/控件 | 14px | 500 | 1.5715× | normal | 表单标签、按钮文字、导航项 |
| 辅助/说明 | 12px | 400 | 1.5715× | normal | 辅助文字、时间戳、元数据 |

### 排版原则
- **14px 为基准**：不同于以 16px 为默认的营销型系统，Pangea 使用 14px 作为标准正文字号，针对数据密集的企业界面优化，屏幕空间寸土寸金。
- **统一行高**：所有文字使用一致的 `行高 = 字号 × 1.5715` 公式，在所有字号下创造可预测的垂直节奏。
- **三级字重**：Regular（400）用于阅读，Medium（500）用于交互/标签元素，SemiBold（600）用于标题。标准用法中不使用 Bold（700）。
- **中文适配**：字体栈在 macOS 上优先使用 PingFang SC，在 Windows 上优先使用 Microsoft YaHei，确保跨平台一致的中文渲染。

## 5. 布局原则

### 间距系统
- 基准单位：4px（含 2px 微调）
- 完整序列：0, 2px, 4px, 6px, 8px, 10px, 12px, 16px, 20px, 24px, 32px, 36px, 40px, 48px, 56px

| 常见用途 | 值 | Token |
|---------|-----|-------|
| 最小间距 | 4px | `@spacing-2` |
| 小间距（图标与文字） | 8px | `@spacing-4` |
| 中间距（控件之间） | 12px | `@spacing-6` |
| 标准内边距 | 16px | `@spacing-7` |
| 区块内边距 | 24px | `@spacing-9` |
| 大区块间距 | 32px | `@spacing-10` |

### 网格与容器
- 标准布局：48px 顶栏 + 200px 侧边栏 + 弹性内容区
- 内容区：白色背景，`border-radius: 8px 0 0 0`
- 页面内容内边距：16px（列表页）、24px（表单页）
- 表单字段网格：`flex-wrap: wrap; gap: 24px`，每个字段 381px 宽（约三列）
- 筛选面板网格：`<a-row :gutter="16">` 配合 `<a-col :span="8">`

### 留白哲学
- **功能性密度**：Pangea 优先考虑信息密度而非装饰性留白。14px 基准字号、13px 表格文字、紧凑的 32px 控件高度，反映出这是一套为处理大量数据的高级用户设计的界面。
- **外壳与内容**：`rgb(242, 243, 245)`（fill-2）外壳（顶栏 + 侧边栏）在白色内容区周围形成一层微妙的灰色框架，建立清晰的图底关系，无需依赖厚重的边框或阴影。
- **纵向弹性**：列表页使用 `height: 100%; display: flex; flex-direction: column; overflow: hidden` 填满可用空间，表格主体独立滚动，而顶栏/筛选区/分页栏保持固定。

### 圆角规范

| Token | 值 | 用途 |
|-------|-----|------|
| `--border-radius-none` | 0 | 无圆角 |
| `--border-radius-small` | 2px | 标签、小徽标 |
| `--border-radius-medium` | 4px | 按钮、输入框、筛选面板 |
| `--border-radius-large` | 8px | 卡片、弹窗、内容区左上角 |
| `--border-radius-circle` | 50% | 头像、圆形图标 |

## 6. 深度与层级

| 层级 | 样式 | 用途 |
|------|------|------|
| 扁平（Level 0） | 无阴影 | 默认状态——卡片、容器、页面表面 |
| 微阴影（Special） | `0 0 1px rgba(0,0,0,0.3)` | 极细的类边框阴影，用于微妙的轮廓定义 |
| 轻阴影（Shadow 1） | `0 0 5px rgba(0,0,0,0.1)` | Popover、Dropdown、Tooltip、轻量卡片悬浮 |
| 中阴影（Shadow 2） | `0 0 10px rgba(0,0,0,0.1)` | Modal、Drawer、重要浮层 |
| 强阴影（Shadow 3） | `0 0 20px rgba(0,0,0,0.1)` | 高优先级对话框、全屏遮罩 |

每级阴影支持 9 个方向变体：`center`、`up`、`down`、`left`、`right`、`left-up`、`left-down`、`right-up`、`right-down`。

**阴影哲学**：Pangea 的阴影系统刻意保持低调。在企业场景中，数十个 UI 元素争夺用户注意力，厚重的阴影只会制造视觉噪音。因此，Pangea 依赖背景填充色阶梯（fill-1 到 fill-4）作为主要的深度指示器，将真正的阴影保留给确实需要传达"浮于页面之上"定位的浮动元素（下拉菜单、弹窗）。所有层级统一使用 `rgba(0,0,0,0.1)` 透明度，保持阴影中性且不突兀——深度仅通过模糊半径（5px → 10px → 20px）来传达。

## 7. 应做与禁忌

### 应做
- 颜色使用本文档第 2 节定义的色值，保持语义一致性（如主文字始终用 `rgb(29, 33, 41)`，不要随意选用相近但不同的灰色）
- 如果在支持 CSS 变量的框架环境中（如 Arco Design Vue），优先使用 `var(--color-*)` 变量引用；在纯 HTML/CSS 或不支持变量的环境中，直接使用对应的 rgb/hex 色值
- 交互元素使用 `rgb(0, 170, 166)`（primary-6）青绿品牌色
- 布局外壳（顶栏 + 侧边栏）背景使用 `rgb(242, 243, 245)`（fill-2）
- 内容区背景使用 `#ffffff`
- 状态指示使用带颜色圆点 + 文字的组合（绿色=成功、橙色=警告、红色=错误、灰色=禁用）
- 每个操作组最多保留一个主按钮
- 所有破坏性操作（删除、禁用等）添加二次确认
- 输入框占位符使用"请输入"，选择器占位符使用"请选择"
- 内容区使用 `border-radius: 8px 0 0 0` 与侧边栏形成视觉分离
- 需要填满剩余高度的内容区使用 `flex: 1; overflow: hidden`

### 禁忌
- 不要使用本文档未定义的颜色——所有色值都应能在第 2 节色彩体系中找到对应
- 不要在同一操作组中放置多个主按钮
- 不要在按钮或输入框上使用大圆角（12px+）——4px 是标准值
- 不要给卡片加重阴影——Pangea 以扁平为主，阴影仅用于浮动层
- 不要省略可折叠侧边栏——所有页面必须在标准布局外壳内渲染
- 不要在页面组件中放置布局结构（顶栏/侧边栏）——页面只渲染内容区
- 不要使用 `outline: none` 移除焦点指示器
- 不要使用 font-weight 700（Bold）——SemiBold（600）是最大字重
- 不要使用 `float` 布局——使用 Flexbox 或 CSS Grid
- 不要在表格行内放置超过 3 个操作按钮——多余的收折到"更多"下拉菜单

## 8. 响应式行为

### 断点

| 名称 | 宽度 | 关键变化 |
|------|------|---------|
| 移动端 | < 768px | 单列布局，侧边栏折叠为 Drawer，紧凑内边距 |
| 平板端 | 768px – 1024px | 侧边栏折叠（48px 仅图标），表单字段两列 |
| 桌面端 | > 1024px | 完整布局——200px 侧边栏展开，表单字段三列 |
| 设计基准 | 1280px | 主要设计目标宽度 |

### 触控目标
- 所有交互控件：最小高度 32px（Arco 默认）
- 移动端触控目标：最小 44×44px 点击区域
- 表格操作链接：使用 `<a-divider direction="vertical">` 分隔，保证足够间距
- 侧边栏折叠按钮：高 40px，占满侧边栏宽度

### 折叠策略
- **侧边栏**：200px 展开 → 48px 折叠（仅图标）→ 移动端变为 Drawer
- **顶部 Tab**：水平模块 Tab → 单模块时隐藏 → 移动端变为汉堡菜单
- **表单字段**：三列（每列 381px）→ 两列 → 移动端单列
- **筛选面板**：桌面端默认展开 → 移动端默认折叠
- **表格**：所有视口均启用水平滚动，`scroll: { x: '100%' }`
- **操作按钮**：行内展示 → 超过 3 个时折叠到"更多"下拉菜单
- **页面内边距**：桌面端 24px → 平板端 16px → 移动端 12px

### 图片行为
- 企业界面以数据驱动为主——图片使用极少
- `@arco-iconbox/vue-pangea-mobile` 图标随字号缩放
- 头像组件使用 Arco 的 `<a-avatar>`，支持响应式尺寸
- 静态资源存放在 `src/assets/`——图标优先使用 SVG 格式
