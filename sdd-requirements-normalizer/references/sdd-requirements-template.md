# SDD Requirement 文档模板

默认使用中文输出。即使某些部分信息较少，也保留章节结构，方便产品、设计与原型协作。

## 1. 需求说明

### 需求目标
- 需求名称（可以是功能 / 页面 / 模块 / 场景名称）
- 一句话概述
- 要解决的问题
- 期望达成的结果

### 目标用户与使用场景
- 主要用户
- 次要用户
- 使用场景：桌面端 / 移动端 / 跨端 / 内部系统 / 外部产品
- 用户从哪里进入该流程或页面

### 范围定义
#### 本期范围
- 本轮原型必须覆盖的能力

#### 非本期范围
- 本轮明确不处理的相邻需求或延伸想法


## 2. 功能列表

| **功能名称** | **功能概述** | **适用角色** | **优先级** | **父记录** |
|------------|------------|------------|----------|----------|
| 功能A       | 功能A的简要描述 | 角色名称    | P0       | -        |
| 功能B       | 功能B的简要描述 | 角色名称    | P1       | 功能A     |

> 优先级说明：P0（高，本期必须上线）、P1（中，本期尽量完成）、P2（低，可延期）
> 每条需求都应写成可验证的行为，而不是抽象目标。


## 3. 功能详细说明

### 核心用户流程
每条流程建议描述：
- 触发条件
- 用户操作步骤
- 系统反馈 / 页面变化
- 流程完成条件


### 交互要求
- 主要操作
- 次要操作
- 选择 / 切换 / 展开 / 收起行为
- 表单交互与校验
- 确认、撤销、删除等高风险操作
- 加载、成功、失败、等待中的反馈
- 返回、跳转、关闭、提交后的行为

### 布局 / 结构要求
- 页面或屏幕整体结构
- 主要区块及其职责
- 信息层级与视觉优先级
- 推荐组件形态：卡片 / 列表 / 表格 / 标签页 / 筛选区 / 侧边栏 / 抽屉 / 弹窗
- 响应式或不同尺寸下的布局变化

### 状态、异常与边界情况
- 空状态
- 错误状态
- 加载状态
- 部分数据状态
- 权限不足或受限状态
- 极端输入或边界条件

## 4. 原型生成要求

### 组件库
- 默认使用 shadcn/ui
- 如果用户明确指定了其他组件库，以用户指定的为准
- 在本节注明所使用的组件库名称

### 主题色
- 主色调：`#00AAA6`
- 其他颜色不做强制约束，由设计阶段根据主色调搭配

### 响应式适配
默认所有页面都需要同时支持 PC 端和移动端，除非用户明确说明仅需支持某一端。对每个主要页面需说明：
- PC 端布局结构
- 移动端布局变化（哪些内容折叠、堆叠、隐藏或转为弹层）
- 表格类内容在小屏下的处理方式（卡片化或横向滚动）
- 操作按钮在小屏下的处理方式（底部固定或收进菜单）
- 导航在小屏下的处理方式（汉堡菜单、底部 Tab 等）

### 通用交互要求
以下交互行为适用于所有页面，无需在每个功能中重复说明，但实现时必须覆盖：
- 加载状态：数据请求期间显示加载指示（骨架屏、Spin 等）
- 空状态：无数据时显示空状态插图和引导文案
- 错误反馈：操作失败时显示明确的错误提示，保留用户已输入内容
- 表单校验：必填字段实时校验，提交时触发全量校验，错误提示显示在对应字段下方
- 防重复提交：提交类按钮在请求期间显示 loading 并禁用
- 危险操作确认：删除、关闭、取消等不可逆操作需二次确认弹窗
- 成功反馈：操作成功后显示全局消息提示

## 5. 验收标准
- 可观察、可评审的检查项
- 同时覆盖功能和交互体验

## 6. 待确认问题 / 默认假设
- 仍需确认的信息
- 为推动原型产出而暂时采用的假设
