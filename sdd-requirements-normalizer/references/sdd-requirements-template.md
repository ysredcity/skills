# SDD Requirement 文档模板

默认使用中文输出。即使某些部分信息较少，也保留章节结构，方便产品、设计与原型协作。

## 1. 产品目标
- 功能 / 页面 / 模块名称
- 一句话概述
- 要解决的问题
- 期望达成的结果

## 2. 目标用户与使用场景
- 主要用户
- 次要用户
- 使用场景：桌面端 / 移动端 / 跨端 / 内部系统 / 外部产品
- 用户从哪里进入该流程或页面

## 3. 范围定义
### 本期范围
- 本轮原型必须覆盖的能力

### 非本期范围
- 本轮明确不处理的相邻需求或延伸想法

## 4. 核心用户流程
每条流程建议描述：
- 触发条件
- 用户操作步骤
- 系统反馈 / 页面变化
- 流程完成条件

## 5. 功能需求
- FR1 ...
- FR2 ...
- FR3 ...

每条需求都应写成可验证的行为，而不是抽象目标。

## 6. 交互要求
- 主要操作
- 次要操作
- 选择 / 切换 / 展开 / 收起行为
- 表单交互与校验
- 确认、撤销、删除等高风险操作
- 加载、成功、失败、等待中的反馈
- 返回、跳转、关闭、提交后的行为

## 7. 布局 / 结构要求
- 页面或屏幕整体结构
- 主要区块及其职责
- 信息层级与视觉优先级
- 推荐组件形态：卡片 / 列表 / 表格 / 标签页 / 筛选区 / 侧边栏 / 抽屉 / 弹窗
- 响应式或不同尺寸下的布局变化

## 8. 状态、异常与边界情况
- 空状态
- 错误状态
- 加载状态
- 部分数据状态
- 权限不足或受限状态
- 极端输入或边界条件

## 9. 非目标
- 明确该原型暂不解决的问题

## 10. 验收标准
- 可观察、可评审的检查项
- 同时覆盖功能和交互体验

## 11. 待确认问题 / 默认假设
- 仍需确认的信息
- 为推动原型产出而暂时采用的假设
