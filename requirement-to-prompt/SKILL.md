# 产品需求转提示词生成器

## 任务目标
- 本 Skill 用于：将不同颗粒度、不同格式的产品需求转化为标准化的提示词模板，供 Claude 大模型生成 demo 原型
- 能力包含：需求颗粒度识别、智能字段补充、结构化提示词生成
- 触发条件：用户需要生成前端原型、需要将产品需求转换为 Claude 可理解的格式、需要补充缺失的需求字段

## 前置准备
- 无需额外依赖，由智能体直接处理

## 操作步骤

### 步骤1：需求分析与颗粒度识别
读取用户提供的原始需求，执行以下分析：

**颗粒度判断**：
- **详细需求**：包含完整的目标描述、设计风格、页面结构、实现要求等 → 直接保留原描述
- **简略需求**：仅包含基本功能描述或简短说明 → 需要智能补充字段

**关键信息提取**：
- 页面类型：列表页、详情页、表单页、仪表盘、弹窗等
- 目标用户：B端、C端、混合
- 核心功能：主要交互流程和操作
- 数据来源：静态数据、API 接口、本地存储

### 步骤2：智能字段补充
对于简略需求，基于场景和模板规范补充缺失字段：

**补充规则**：
1. **目标描述**：根据页面类型补充标准目标模板
2. **设计风格**：根据目标用户补充默认风格（B端→企业风，C端→电商/社交风）
3. **页面结构**：根据页面类型补充标准区块结构
4. **实现要求**：补充响应式适配、技术栈规范

**优先级原则**：
- 如果需求中有明确描述 → 严格按照原描述
- 如果需求描述模糊 → 基于模板规范补充合理默认值
- 如果需求未提及 → 补充标准最佳实践

参考 [references/granularity-guide.md](references/granularity-guide.md) 了解详细的颗粒度识别方法。
参考 [references/page-type-templates.md](references/page-type-templates.md) 获取常见页面类型的字段补充模板。

### 步骤3：生成结构化提示词
按照以下标准格式组织输出：

```
目标：<清晰描述页面目标和核心价值>

## 设计风格
<整体风格描述>
- <色调选择>
- <字体规范>
- <组件风格>

## 页面结构
**<区块1名称>**
<详细描述，包括字段、交互、响应式规则>

**<区块2名称>**
<详细描述，包括字段、交互、响应式规则>

## 实现要求
- <技术实现要求>
- <交互细节要求>
- <数据模拟方式>
```

**格式要求**：
- 目标描述：单行，明确用户场景和核心价值
- 设计风格：3-5个要点，包含色调、字体、组件规范
- 页面结构：按区块分层，每个区块包含字段定义、交互规则、响应式适配
- 实现要求：技术栈、交互逻辑、数据处理方式

### 步骤4：验证与优化
检查生成的提示词是否符合以下标准：

**完整性检查**：
- [ ] 包含目标、设计风格、页面结构、实现要求四个核心部分
- [ ] 每个页面区块都有明确的字段定义和交互规则
- [ ] 响应式适配规则清晰（PC端/移动端差异化说明）

**合理性检查**：
- [ ] 字段补充符合业务场景
- [ ] 技术实现方案可行
- [ ] 交互逻辑完整且可测试

**格式检查**：
- [ ] 符合用户提供的示例格式
- [ ] Markdown 层级清晰（H2标题加粗，H3普通加粗）
- [ ] 列表项使用正确的编号和缩进

参考 [assets/template-example.md](assets/template-example.md) 查看完整的标准示例。

- 对于模糊需求，提供最合理的默认方案，并标注"可根据实际情况调整"
- 保持提示词的结构化程度，方便 Claude 理解和执行

## 使用示例

### 示例1：详细需求转换
**输入**（用户提供的企业级需求评审审批详情页完整需求）

**处理**：
1. 识别为详细需求，包含完整的四个部分
2. 直接保留原描述，无需补充
3. 格式化为标准模板输出

**输出**：完全按照用户提供的示例格式生成

### 示例2：简略需求补充
**输入**："需要一个商品列表页，支持搜索和筛选，B端用户使用"

**处理**：
1. 识别为简略需求
2. 补充目标：生成一个企业级商品管理列表页，面向B端运营人员，支持PC与移动端响应式适配
3. 补充设计风格：B端企业风，蓝灰色主色调，表格+卡片布局
4. 补充页面结构：搜索筛选区、列表数据区、分页区、操作栏
5. 补充实现要求：响应式适配，支持分页、排序、批量操作

**输出**：生成包含完整四个部分的结构化提示词

### 示例3：模糊需求处理
**输入**："做一个用户管理页面"

**处理**：
1. 识别为极度简略需求
2. 补充目标：生成一个基础的用户管理页面，包含列表展示、新增编辑、删除功能
3. 补充设计风格：标准B端风格，简洁专业
4. 补充页面结构：用户列表、搜索区、操作区、弹窗表单
5. 补充实现要求：基础CRUD功能，包含表单验证和交互反馈
6. 标注注：当前为最小可用版本，可根据实际业务需求扩展字段和功能
