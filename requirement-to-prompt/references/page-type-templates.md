## 列表页模板

### 适用场景
- 数据展示类页面（商品列表、用户列表、订单列表等）
- 支持筛选、排序、分页的数据浏览页面

### 标准目标描述
"生成一个[业务对象]管理列表页，面向[目标用户]，支持PC与移动端响应式适配"

### 设计风格
```
整体采用 B端/ C端管理系统风格：克制/活泼、专业/友好
主色调使用[色系]，避免过于花哨的视觉效果
字体选用 PingFang SC / HarmonyOS Sans 等中文友好字体
组件参考企业级设计规范，表格间距统一，层级清晰
移动端优先折叠非核心信息，保证核心操作区始终可见
```

### 页面结构
```
**1. 搜索筛选区**
PC端：横向排列，最多3个主要筛选条件，其余可收起
移动端：默认仅展示1个筛选条件，其余通过「更多筛选」折叠
常用筛选：关键字搜索、分类筛选、状态筛选、时间范围

**2. 列表数据区**
PC端：表格展示，支持列排序，行悬停效果
移动端：卡片列表，每卡展示核心字段3-4个
列表字段：[字段1]、[字段2]、[字段3]、[状态]、[时间]
长文本默认折叠超过2行，提供「展开」

**3. 操作区**
顶部操作栏：「新增」主按钮 + 「导出」次按钮
行内操作：「编辑」和「删除」操作按钮
批量操作：支持多选后批量删除/批量导出

**4. 分页区**
PC端：底部分页器，显示总数、每页条数选择器
移动端：底部「加载更多」按钮或无限滚动
```

### 响应式规则
- PC端：表格固定列，横向滚动支持
- 移动端：卡片展示，隐藏次要字段

## 详情页模板

### 适用场景
- 查看单个数据对象完整信息的页面
- 通常包含多个信息区块（基本信息、关联信息、操作记录等）

### 标准目标描述
"生成一个[业务对象]详情页，面向[目标用户]，支持PC与移动端响应式适配"

### 设计风格
```
整体采用 B端企业风格：信息密度适中、层次清晰
主色调使用蓝灰色系，重点信息使用强调色突出
字体选用中文友好字体，标题使用较大字号
组件参考卡片式设计，区块间距统一
移动端优先折叠非核心信息，保留操作按钮可见性
```

### 页面结构
```
**1. 页头区域（只读）**
PC端：以 2×4 网格展示所有字段
移动端：默认仅展示核心字段，其余折叠显示
字段包含：[主键字段]、[关联字段1]、[关联字段2]、[状态]、[时间]

**2. 内容详情区域（只读）**
分为多个信息区块，通过 Tab 或分组卡片区分
区块1：基本信息（字段A、字段B、字段C）
区块2：关联信息（关联列表、相关数据）
区块3：历史记录（时间线展示）
长文本默认折叠超过3行，提供「展开全文」

**3. 操作区域**
固定在页面底部或右侧悬浮
操作按钮：[编辑]、[删除]、[返回]
重要操作需二次确认
```

### 响应式规则
- PC端：左右分栏或卡片网格布局
- 移动端：垂直堆叠，信息区折叠

## 表单页模板

### 适用场景
- 新增或编辑数据对象的页面
- 包含多个表单字段和验证规则

### 标准目标描述
"生成一个[业务对象]表单页，面向[目标用户]，支持PC与移动端响应式适配"

### 设计风格
```
整体采用清晰的表单风格：字段对齐、间距统一
主色调用于必填标记和操作按钮
字体选用中文友好字体，标签和输入框对比明显
组件参考企业级表单规范，表单项分组清晰
移动端优化输入体验，避免频繁键盘弹出
```

### 页面结构
```
**1. 表单主体区域**
按功能分组，每组 3-5 个字段
必填字段使用红色星号标记
表单字段：[文本输入]、[下拉选择]、[日期选择]、[文件上传]
复杂字段提供帮助文字或示例

**2. 操作按钮区域**
底部固定或右对齐
主按钮：「提交」或「保存」
次按钮：「取消」或「返回」

**3. 验证提示区域**
实时校验：失焦时触发
提交校验：点击提交时触发所有字段验证
错误提示：红色文字显示在字段下方
```

### 响应式规则
- PC端：表单项横向排列，最大化利用宽度
- 移动端：单列布局，优化触摸目标大小

## 仪表盘模板

### 适用场景
- 数据分析和监控页面
- 包含多个图表和关键指标展示

### 标准目标描述
"生成一个[业务领域]数据仪表盘，面向[目标用户]，支持PC与移动端响应式适配"

### 设计风格
```
整体采用数据可视化风格：卡片化、图表丰富
主色调用于趋势指示（上涨绿色、下跌红色）
字体选用等宽字体显示数值，提高可读性
组件参考 BI 工具设计规范，图表类型合理
移动端优先展示核心指标，详细图表可展开查看
```

### 页面结构
```
**1. 顶部指标区**
PC端：4个关键指标卡片横向排列
移动端：2个指标卡片，支持横向滑动
指标包含：[指标名]、[数值]、[环比趋势]、[同比趋势]

**2. 图表区域**
PC端：2×2 网格布局，展示4个主要图表
移动端：单列布局，图表可点击放大
图表类型：折线图（趋势）、柱状图（对比）、饼图（占比）

**3. 数据列表区**
展示详细数据表格，支持排序和筛选
移动端：卡片列表或简化表格

**4. 筛选控制区**
时间范围选择、维度切换、导出功能
移动端：折叠在「筛选」按钮中
```

### 响应式规则
- PC端：多图表网格布局，充分利用空间
- 移动端：单列布局，图表可交互查看详情

## 弹窗页模板

### 适用场景
- 弹窗形式展示的轻量级表单或信息页
- 不独立占用页面空间

### 标准目标描述
"生成一个[业务对象]弹窗，面向[目标用户]，支持PC与移动端响应式适配"

### 设计风格
```
整体采用简洁风格：内容聚焦、操作明确
遮罩层半透明黑色，弹窗居中显示
字体选用中文友好字体，字号适配弹窗尺寸
组件参考模态框规范，支持关闭和确认操作
移动端：弹窗宽度接近全屏，便于操作
```

### 页面结构
```
**1. 弹窗标题栏**
左侧：[标题]
右侧：关闭图标按钮

**2. 弹窗内容区域**
内容高度自适应，超出时显示滚动条
表单字段或信息展示

**3. 弹窗操作栏**
底部固定，右对齐
主按钮：「确认」
次按钮：「取消」
```

### 响应式规则
- PC端：弹窗宽度 600-800px，居中显示
- 移动端：弹窗宽度 90%vw，底部对齐或全屏

## 使用说明

### 选择模板的原则
1. 优先级：明确类型 > 功能描述 > 默认列表页
2. 混合场景：主要功能对应的页面类型
3. 特殊场景：基于最接近的模板调整

### 补充字段的策略
1. **业务对象字段**：根据业务上下文补充合理的字段名称
2. **状态字段**：补充常见状态（草稿、已发布、已下架等）
3. **时间字段**：补充创建时间、更新时间、操作时间等
4. **用户字段**：补充创建人、更新人、操作人等

### 标注补充来源
对于补充的内容，在末尾添加标注：
```
（注：以上字段为基于[页面类型]的默认补充，可根据实际业务调整）
```
