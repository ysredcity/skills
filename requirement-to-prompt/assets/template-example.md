目标：生成一个企业级需求评审审批详情页，面向B端用户，支持PC与移动端响应式适配。

## 设计风格

整体采用 B端企业管理系统风格：克制、专业、信息密度适中
主色调使用蓝灰色系（如 #00AAA6 主色 + #F2F3F5 背景），避免过于花哨的视觉效果
字体选用 PingFang SC / HarmonyOS Sans 等中文友好字体，英文辅以 IBM Plex Sans
组件参考 shadcn/ui 设计规范，卡片间距统一，层级清晰
移动端优先折叠非核心信息，保证核心操作区始终可见

## 页面结构

**1. 页头区域（只读）**

PC端：以 2×4 网格展示所有字段（需求名称单独占一行，字号加大突出显示）
移动端：默认仅展示「需求名称、提报人、提报时间、评审类型」，其余字段通过「展开详情」按钮折叠显示，展开/收起有过渡动画
字段包含：需求名称（H2级标题）、需求提报人、提报时间、关联产品、产品经理、所属PDT、PDT经理、评审类型

**2. 评审内容区域（只读）**

分为「需求原声」和「需求分析」两个子版块，通过 Tab 或分组卡片区分
需求原声：背景与痛点、核心诉求、业务价值与目标、期望完成时间
需求分析：需求功能逻辑、UAT验收标准、需求规模、需求分类、评审专家
所有字段为只读展示，长文本内容默认折叠超过3行部分，提供「展开全文」

**3. 评审记录区域**

展示为表格列表（PC端），移动端降级为卡片列表
列表字段：评审时间、评审人、角色标签（仅显示产品经理 / SE / TSE / PDT经理）、评审意见（截断显示）、个人决议（用色块标签区分：同意=绿色、需修改=橙色、驳回=红色）
操作列：「查看」按钮，点击弹出模态对话框，展示该条评审记录的完整表单字段（只读）

**4. 评审意见区域（可交互）**

该区域为页面核心交互区，固定在页面下方或单独卡片展示。

**辅助操作**：

顶部放置「✨ AI 辅助评估」按钮（次要但醒目），点击后调用大模型 API，流式返回评估结果并自动填充下方4个评分字段和「评审意见」输入框，填充过程有 loading 动效（逐字输入效果）

**表单字段（4个评分 + 2个输入）**：

需求合规性 — 1~5星评分组件，下方显示浅色帮助文字：「是否符合公司战略、IT规划、信息安全、数据合规要求」
必要性与合理性 — 1~5星评分，帮助文字：「是否真的解决业务问题，是否重复建设或存在逻辑矛盾」
资料完整性 — 1~5星评分，帮助文字：「需求背景、目标、用户、场景是否写清楚」
基本可行性 — 1~5星评分，帮助文字：「技术、资源、周期是否存在硬性障碍」
评审意见 — 多行文本框（至少4行），必填，未填写时提交触发红色错误提示
个人决议 — 单选按钮组（同意 / 需修改 / 驳回），默认选中「同意」；不同选项对应不同强调色（绿/橙/红）

**提交按钮**：

校验通过后：新增评审记录 → 刷新评审记录列表 → 调用「待办转已办」回调接口 → 显示成功提示
校验失败：高亮必填项并滚动至第一个错误字段

## 实现要求
- 响应式适配：支持响应式适配，确保在PC、移动端中都能够正常操作和显示页面内容
- 技术实现：单文件 HTML + Tailwind CDN + 原生 JS（无需框架）。mock 数据直接写在 JS 中，所有交互逻辑完整实现（AI 辅助评估可用 setTimeout 模拟流式效果）。
